# 00 · Design Overview

## Связь с GDD

`GDD/` — концептуальный документ: пиллары, целевая аудитория, описание систем верхнего уровня. `design/` — продуктовая детализация под Godot-реализацию: конкретные числа, экраны, переходы, балансы, шаблоны.

При расхождении — `GDD/` главнее. Если в `design/` появилось что-то, чего нет в GDD, и это влияет на видение — синхронизировать GDD.

## Принципы дизайн-решений

1. **MVP first.** Любое решение проверяется вопросом «нужно ли это для MVP?» Если нет — выкидываем в backlog.
2. **Data-driven.** Контент — `.tres`, балансы — числа, не код. Дизайнер может править без программиста.
3. **Mobile UX.** Тач-таргеты, безопасные зоны, портретная ориентация — всегда.
4. **Анти-paywall.** F2P достигает 90% эффективности. Каждый платный оффер должен соответствовать.
5. **Ясность над «крутостью».** Если игрок не понимает за 5 секунд — упрощаем.

## Конвенции документов

- Числа — с единицами измерения
- Балансы — в таблицах, чтобы видеть прогрессию
- UX-флоу — wireframe ASCII или ссылка на Figma (когда появится)
- Решения с большими последствиями — ADR-блок: `> ADR-NNN: <решение>. **Дата:**. **Причина:**.`
